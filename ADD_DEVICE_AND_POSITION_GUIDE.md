# PositionSettingsView 添加设备和位置功能 - 使用指南

## ✅ 功能概述

在 **PositionSettingsView**（位置设置界面）中现在可以：

1. ✅ **添加新设备** - 支持 12 种设备类型
2. ✅ **添加新位置** - 为任何设备添加工作位置
3. ✅ **删除位置** - 删除不需要的位置
4. ✅ **双向同步** - 与 DeviceDebugView 自动同步

## 🎯 支持的设备类型

| 设备类型 | 连接方式 | 主要用途 |
|---------|---------|---------|
| CAN 电机 | CAN 总线 | 运动控制 |
| EtherCAT 电机 | EtherCAT | 高精度运动控制 |
| 注射泵 | 串口 | 液体注射 |
| 蠕动泵 | 串口 | 液体输送 |
| 自定义泵 | CAN 总线 | 自定义液体控制 |
| 离心机 | 串口 | 样品离心 |
| TCU 温控 | 串口 | 温度控制 |
| 冷水机 | 串口 | 冷却控制 |
| 称重传感器 | 串口 | 重量测量 |
| 扫码枪 | 网络 | 条码扫描 |
| Jaka 机器人 | 网络 | 机器人控制 |
| IO 设备 | EtherCAT | 数字输入输出 |

## 📖 使用方法

### 1. 添加新设备

**步骤：**

1. 打开 **PositionSettingsView**（位置设置界面）
2. 点击顶部工具栏的 **"添加设备"** 按钮（绿色按钮）
3. 在弹出的对话框中：
   - 选择**设备类型**（必选）
   - 输入**设备 ID**（必填，唯一标识符）
   - 输入**设备名称**（必填）
   - 输入**设备描述**（可选）
   - 根据设备类型配置**连接参数**：
     - **CAN 设备**：输入 CAN 节点 ID
     - **串口设备**：选择串口号、设置波特率
     - **网络设备**：输入 IP 地址和端口号
     - **EtherCAT 设备**：输入从站 ID
4. 点击 **"添加"** 按钮
5. ✅ 设备添加成功，可以在筛选列表中看到新设备

**示例：添加一个 CAN 电机**
```
设备类型: CAN 电机
设备 ID: MOTOR_X
设备名称: X轴电机
设备描述: X轴运动电机
CAN 节点 ID: 1
```

**示例：添加一个离心机**
```
设备类型: 离心机
设备 ID: CENTRIFUGE_01
设备名称: 离心机1号
设备描述: 实验室离心机
串口号: COM3
波特率: 9600
```

### 2. 添加新位置

**方式一：为现有设备添加位置**

1. 点击顶部工具栏的 **"添加位置"** 按钮
2. 系统自动创建一个新位置（使用第一个可用设备）
3. 在位置列表中选择新添加的位置
4. 修改位置参数：
   - **位置名称**：修改为有意义的名称
   - **位置值**：设置实际位置值
   - **速度**：设置运动速度
5. 点击 **"保存配置"** 按钮
6. ✅ 位置已保存并同步到 DeviceDebugView

**方式二：为新添加的设备添加位置**

1. 先添加一个新设备（见上文）
2. 点击 **"添加位置"** 按钮
3. 如果想为特定设备添加位置，需要先修改新位置的设备关联
4. 修改位置参数
5. 保存配置

### 3. 删除位置

1. 在位置列表中选择要删除的位置
2. 点击 **"删除"** 按钮
3. 确认删除操作
4. ✅ 位置已删除并同步到 DeviceDebugView

### 4. 筛选位置

**按设备筛选：**
- 在顶部的 **"Filter"** 下拉框中选择设备名称
- 位置列表自动过滤，只显示该设备的位置

**搜索位置：**
- 在搜索框中输入关键字
- 自动搜索设备 ID、设备名称、位置名称

## ⚙️ 连接参数配置

### CAN 设备（CAN 电机、自定义泵）
- **CAN 节点 ID**：设备在 CAN 总线上的节点编号

### 串口设备（注射泵、蠕动泵、离心机、TCU、冷水机、称重传感器）
- **串口号**：COM1 ~ COM8
- **波特率**：默认 9600，可选 115200、57600 等

### 网络设备（扫码枪、Jaka 机器人）
- **IP 地址**：设备的网络 IP 地址（如 192.168.1.100）
- **端口号**：通信端口号

### EtherCAT 设备（EtherCAT 电机、IO 设备）
- **从站 ID**：EtherCAT 从站编号

## 💡 使用技巧

### 1. 设备 ID 命名规范
建议使用有意义的命名规范：
```
MOTOR_X        - X 轴电机
MOTOR_Y        - Y 轴电机
PUMP_01        - 1号泵
CENTRIFUGE_01  - 1号离心机
ROBOT_01       - 1号机器人
```

### 2. 批量添加位置
```
1. 添加新设备
2. 重复点击"添加位置"创建多个位置
3. 统一修改位置参数
4. 一次性保存
```

### 3. 复制设备配置
```
1. 导出当前配置（Export）
2. 手动编辑 JSON 文件，复制设备
3. 修改设备 ID 和名称
4. 导入修改后的配置（Import）
```

### 4. 验证设备 ID 唯一性
系统会自动检查设备 ID 是否已存在，如果存在会提示错误：
```
设备 ID 'MOTOR_X' 已存在，请使用其他 ID。
```

## 📊 设备默认参数

添加设备时，系统会自动设置默认参数：

### CAN 电机
- Unit: mm
- JogSpeed: 100

### EtherCAT 电机
- Unit: mm
- JogSpeed: 100

### 注射泵
- SyringeVolume: 50 ml
- LiquidOffset: 0

### 蠕动泵
- MaxRPM: 600
- MaxFlowRate: 100 ml/min
- Description: 可自定义

### 自定义泵
- PulsePerRevolution: 200
- GearRatio: 1.0
- MaxRPM: 600

### 离心机
- BaudRate: 9600
- 默认添加空的 WorkPositions 列表

### 称重传感器
- MaxWeight: 1000 g
- DecimalPlaces: 2

### 其他设备
- 根据设备类型设置合理的默认值

## ⚠️ 注意事项

### 1. 设备 ID 必须唯一
每个设备的 ID 在系统中必须唯一，否则无法添加。

### 2. 保存后才能同步
添加设备或位置后，**必须点击"保存配置"才能同步到 DeviceDebugView**。

### 3. 连接参数根据实际情况配置
- 串口设备：确保串口号和波特率正确
- 网络设备：确保 IP 地址和端口可访问
- CAN/EtherCAT：确保节点 ID 不冲突

### 4. 删除设备
目前不支持直接删除设备，如需删除设备：
- 方式一：手动编辑配置文件
- 方式二：导出配置，编辑 JSON，重新导入

### 5. 设备参数修改
添加设备后，如需修改详细参数：
- 保存并导出配置
- 编辑 JSON 文件
- 重新导入配置

## 🔄 同步机制

### 添加设备
```
PositionSettingsView
    ↓
添加设备到 CurrentConfig
    ↓
更新设备筛选列表
    ↓
（暂不自动同步到 DeviceDebugView）
    ↓
保存配置 → 重新导入配置 → 同步完成
```

### 添加位置
```
PositionSettingsView
    ↓
添加位置到 AllPositions
    ↓
添加位置到 CurrentConfig
    ↓
发布 PositionAddedEvent
    ↓
DeviceDebugView 自动同步
```

## 🎓 完整示例流程

### 示例：为新实验室配置设备

**第一步：添加电机设备**
```
1. 点击"添加设备"
2. 选择"CAN 电机"
3. DeviceId: MOTOR_X
4. Name: X轴电机
5. CAN节点ID: 1
6. 点击"添加"
```

**第二步：为电机添加工作位置**
```
1. 点击"添加位置"
2. 修改位置名称: "取样位置"
3. 修改位置值: 100
4. 修改速度: 200
5. 点击"保存配置"
```

**第三步：继续添加更多位置**
```
1. 重复点击"添加位置"
2. 修改参数：
   - "放置位置", 200, 200
   - "等待位置", 0, 100
   - "维护位置", 300, 50
3. 点击"保存配置"
```

**第四步：添加离心机**
```
1. 点击"添加设备"
2. 选择"离心机"
3. DeviceId: CENTRIFUGE_01

```

**第五步：验证配置**
```
1. 切换到 DeviceDebugView
2. 查看设备列表，确认新设备已同步
```

## 🔍 故障排除

### 问题：点击"添加设备"无反应
**原因：** 未导入配置  
**解决：** 先导入一个配置文件，或在 DeviceDebugView 中导入配置

### 问题：设备 ID 冲突
**原因：** 设备 ID 已存在  
**解决：** 使用其他唯一的设备 ID

### 问题：添加设备后在调试界面看不到
**原因：** 未保存配置或未同步  
**解决：** 
1. 点击"保存配置"
2. 在 DeviceDebugView 重新导入配置
3. 或重启应用程序

### 问题：添加的位置没有保存
**原因：** 忘记点击"保存配置"  
**解决：** 修改位置参数后必须保存

### 问题：无法删除设备
**说明：** 当前版本不支持直接删除设备  
**解决：** 导出配置，手动编辑 JSON 文件，删除设备后重新导入
